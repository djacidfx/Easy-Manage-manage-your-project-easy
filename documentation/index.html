Getting Started

Easy Manage is a React app for managing tasks of a certain project, it comes with a beautiful user interface and tools that will help you manage your project easily.


Recuirments

Make sure you have nodejs and npm installed in your device

You can check for them after installation by writing npm -v and node -v in the cmd

install node

npm -v
9.1.3      npm version we used

node -v
v18.12.1     nodejs version we used


Install npm Packages

Open App folder directory (main_file/easy-manage) by a code editor like vscode and write in the terminal npm install to install template packages

npm install


Development server

After instalation your can run npm run dev command to see the website alive (http://localhost:5173/easy-manage)

npm run dev

Changing Tasks

Tasks that existed in the tasks.json file are fetched when there are no tasks in localstorage.
When you use the application and add tasks yourself the data in the tasks.json file will not be called when the localstorage already has tasks.
If you want to integrate backend API you should understand this part well.
If you will use his application as it's you can cancel this part.

Here you will find team information used in the Application




Here are the types of keys you can use to add new tasks
i.e: This is Task Schema

{
  "id": string, => unique identifer string
  "title": string,
  "type": TaskTypes, => TaskTypes is existed in types folder as we said in Jobs type
  "description": string,
  "developers": string[], => Array of different members id
  "estimatedTime": number, => number of seconds to finish the task (will be converted to days)
  "startingTime": number, [not existed in todo type task] => date of starting the task expressed by seconds
  "endingTime": number, => [existed in done type task] => date of ending the task expressed by seconds
  "rating": Rating, => a number expressed the rate of task
}

We used three types of tasks for our application as is clear in the app
   todo tasks           : that are planned to be done but not in working
   progress tasks : tasks that are in the working area
   done tasks          : tasks that are completed


rating is a number (0, ∞) which determines the review of the state of the task
   r < 1: task takes more time to be completed than estimated time (bad rate)
   r = 1: task takes the exact same time as estimated time (normal rate)
   r > 1: task completed in less time than estimated time (good rate)

Backend Integration

If you want to integrate backend API and use it as a full-stack web app it's easy but you should take some points into consideration

  • You should provide Shemas matches with the app data (tasks and team), We discussed this in coustomization
  • Make API routes matches all process done to tasks (CRUD Operations)
  • Fetch API routes beside redux state management reducers

In the app/index.tsx file you will find how to grab the data from our data folder or from localstorage and we put comments to make the process clear for you if you want to make an edit.

You will add the fetch requests to your API and save the data in redux store as it is written

When operating on tasks such as adding, updating or deleting the update take place in two places which are local storage and redux state management
You will have to fetch your API request also to update the data in your database after changing it locally

Updating state takes place via dispatch function, we have four different functions to change out tasks as its in tasksSlice.tsx file
You will have to fetch API requests with the same functionality as the dispatch function called, to update your database too.

Uploading to the Server

Once you finised coustomization and want to upload the website to the internet you can do the following steps

Note: you need to do some changes before building Template

  • -> Change the homepage property to your website link from package.json file
  • -> You need to change the route basename if you changed the directory of the website (easy-manage)

  • -> Goto src/app/index.tsx and change easy-manage to the directory you wrote before in the package.json
    If there is no directory then make the base name equals{/}

After this write npm run build in the cmd

npm run build

You will find build folder created and this folder is the folder you will upload to your hosting server


In case of private shared hosing
Make folder in public_html named by the same name of the directory you make in package.json file and extract dist files in it Note: dont put the dist folder itself put the files inside it in the folder you created


Note
Uploading to server differes from on provider to another and you may need to do more modifications to the app before building it to be ready to be uploaded so we will leave a guide from vite documentation on how to upload on different providers.
